# UX Quality Bar

Use this reference when defining or evaluating whether a milestone is ready for users.

## Five-star standard

A consumer-grade experience should feel:

- clear on first use
- fast enough to maintain trust
- visually coherent
- forgiving when the user makes mistakes
- consistent across primary flows
- polished enough that rough edges do not distract from the value

## Core UX gates

Before calling work ready, check:

- The primary user can complete the main job without guidance.
- The UI hierarchy makes the next action obvious.
- Labels, buttons, and empty states use plain language.
- Error states explain what happened and how to recover.
- Loading, success, and failure feedback are visible.
- Defaults are sensible and reduce user effort.
- Mobile and desktop experiences both support the core flow.
- Accessibility problems do not block comprehension or action.
- The product feels internally consistent in spacing, copy tone, and interaction patterns.

## Common ways teams miss the bar

- Shipping broad functionality with weak onboarding
- Leaving placeholder copy or unclear labels in core flows
- Handling the happy path only
- Ignoring empty states, retries, and failures
- Treating visual consistency as optional
- Letting setup friction obscure the main value

## Quality review prompts

Ask:

- Where will a first-time user hesitate?
- What part of the flow still feels manual, confusing, or fragile?
- Which edge state will most damage trust if handled poorly?
- What should be simplified before anything new is added?
- Does this feel intentionally designed, or merely assembled?

## Release recommendation

If the experience is functional but not yet trustworthy, coherent, or low-friction, recommend reducing scope rather than declaring it ready.
