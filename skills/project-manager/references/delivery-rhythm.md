# Delivery Rhythm

Use this reference when planning weekly execution, writing status updates, or keeping a project moving without losing focus.

## Weekly operating cadence

Use a simple rhythm:

1. Reconfirm the milestone outcome
2. Recheck the cut line
3. Identify the top blockers and decisions
4. Set the week's must-finish items
5. Review status against quality bar, not just task completion

## Replan triggers

Replan when:

- the core flow is not meeting the UX bar
- the milestone gained unplanned scope
- a dependency moved
- a blocker threatens the date
- the team is completing tasks but not converging on a polished outcome

## Delivery discipline prompts

Use these prompts to keep the team honest:

- "What are we protecting in this milestone?"
- "What can we cut without harming the core experience?"
- "What looks done on paper but still feels weak to a user?"
- "What would make this feel five-star instead of merely functional?"
